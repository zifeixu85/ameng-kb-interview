# 已有 Vault 接入

## 默认只读

第一次接入只列出顶层目录、入口文件与现有规则，不读取全部正文，也不执行迁移。

## 先做职责映射

例如用户已经有：

```text
Projects/      → 30-项目工作区
Resources/     → 40-学习输入库
Cards/         → 20-可复用资产库
```

把这些映射写进结构方案，确认后复用现有目录。不要为了统一命名而立即改名。

## 冲突策略

- 同名目录存在：复用。
- 同名文件存在：跳过。
- 已有文件内容与模板不同：保留用户版本。
- 需要补充采访内容：新建 `标题-采访补充-YYYY-MM-DD.md`。
- 迁移、合并、删除：单独列计划，再次确认。

## 推荐提示词

```text
请使用 $ameng-kb-interview。这是一个已有内容的 Vault。
先只读检查顶层目录和入口文件，不读取私人正文，不移动、改名、覆盖或删除。
请先给我兼容映射方案和准备新增的文件清单，等我确认后再继续。
```
