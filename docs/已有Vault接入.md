# 已有 Vault 接入

## 证据优先，默认只读

第一次接入先读取现有规则、目录、MOC / README / 索引，再按用户的建库目标渐进式读取相关 Markdown 正文。已有笔记就是采访证据；包含 `private` 内容不意味着必须从零重问。

整个阶段保持只读，但“只读”不等于“只看目录”。用户确认映射和写入边界前，不创建、移动、改名、覆盖或删除任何内容。

## 先生成已知上下文

开始补问前，先输出：

- 已知信息：可直接复用的结论与来源；
- 待确认：可能过期或只有一个旧来源；
- 冲突：不同笔记说法不一致；
- 缺失：现有笔记确实无法回答。

只对后三类做确认或补问。用户不需要把已经写过的人生经历、项目背景和方法重述一遍。

## 先做职责映射

例如用户已经有：

```text
Projects/      → 30-项目工作区
Resources/     → 40-学习输入库
Cards/         → 20-可复用资产库
```

把这些映射写进结构方案，确认后复用现有目录。不要为了统一命名而立即改名。

## 冲突策略

- 同名目录存在：复用。
- 同名文件存在：跳过。
- 已有文件内容与模板不同：保留用户版本。
- 需要补充采访内容：新建 `标题-采访补充-YYYY-MM-DD.md`。
- 迁移、合并、删除：单独列计划，再次确认。

## 推荐提示词

```text
请使用 $ameng-kb-interview。这是一个已有内容的 Vault。
请先只读读取现有规则、目录和与建库目标相关的笔记正文，包括必要的 private 笔记。
先整理已知信息、待确认、冲突和缺失，已有答案不要重新采访。
不移动、改名、覆盖或删除任何现有内容。请先给我兼容映射、可复用笔记和准备新增文件的清单，等我确认后再写入。
```
