# 目录设计说明

## 为什么不直接复制 A梦的完整 Vault

A梦当前 Vault 里存在历史迁移、客户项目、黑客松、内容生产和业务协作等长期积累。这些目录对 A梦有用，但对一个刚开始的人不是通用前提。

公开版只抽象出六个稳定职责，并让具体模块由采访结果决定。

## 固定层

```text
00-系统工作台/
10-自我说明书/
20-可复用资产库/
30-项目工作区/
40-学习输入库/
99-附件库/
```

固定的是信息生命周期和隐私边界，不是某个职业的业务分类。

## 自我说明书不是四个粗分类

公开版采用 A梦真实知识库的完整自我说明书地图：

```text
01-身份与经历
02-价值观与驱动力
03-性格底色与心理测试
04-偏好审美与表达习惯
05-身体健康与能量状态
06-家庭亲密关系与责任
07-社会生活与重要关系
08-状态变化与阶段复盘
09-日记与自我观察
10-AI协作偏好
99-隐私边界与公开授权
```

`99` 固定存在；其余是可渐进启用的自我模块。标准版优先检查身份、价值观、偏好、阶段和 AI 协作，深度版才提供完整采访地图。心理、健康、家庭、关系和日记等敏感模块必须明确同意后启用。

逻辑上保留完整方法论，物理上不制造空目录。

## 动态层

| 模块 | 只有出现什么证据才启用 |
|---|---|
| 对外角色与表达 | 用户会持续对外介绍、写作或经营个人品牌 |
| 产品与服务 | 已经提供产品、课程、咨询或服务 |
| 客户与用户 | 有访谈、问题、异议和使用证据 |
| 素材与案例 | 已提取出可复用的故事、案例、原话或结果 |
| 方法与判断 | 已形成可复述、可验证的方法或判断 |
| 内容生产 | 有持续平台、受众和发布流程 |
| 业务知识 | 有重复 FAQ、SOP、服务流程和人工接管边界 |
| 合作与机会 | 需要管理线索、承诺和下一次联系 |
| 主题分享 | 同一主题会复讲、迭代与反向沉淀 |
| 职业资产 | 求职、晋升、作品集或绩效证明是核心目标 |

项目类型同样按需出现。没有参加黑客松，就不会生成黑客松目录。

## 三条防膨胀规则

1. 没有真实文件，不创建模块。
2. 状态写在属性里，不建立“进行中 / 已完成”文件夹。
3. 资产按主题和用途组织，来源写进属性，不按书、课、老师建立资产目录。

## 写入顺序

判断一条内容去哪里时，依次判断：

1. 隐私级别；
2. 它是原始输入、项目过程还是可复用资产；
3. 它属于身份、故事、方法、产品、客户、内容还是其他对象；
4. 最后才按主题建立子目录或标签。

这个顺序能避免“看起来分类正确，但 private 内容被公开使用”的问题。
