# 隐私说明

## 数据存在哪里

本仓库自身不提供云服务、账号系统、遥测或后台。初始化脚本只在你指定的本地 Vault 中创建 Markdown 与目录，不主动联网。

但当你使用 Codex 或其他 AI Agent 处理采访和笔记时，被选中的内容可能会发送给你配置的模型服务商。请根据所使用产品的隐私条款判断是否适合处理敏感信息。

`sensitivity: private / internal / public` 与 `AGENTS.md` 是 AI 协作策略，不是加密、操作系统访问控制或模型隔离。笔记仍是明文 Markdown：Codex、本机其他有权限的用户或应用可能读取，iCloud、Obsidian Sync、Dropbox 等同步工具可能上传整个 Vault。

如果你需要硬隔离，请使用独立、不同步或经加密保护的 Vault，并避免向 AI 提供密码、Token、身份证件或其他秘密。

## 默认保护

- `10-自我说明书` 默认 private。
- 日记、健康、家庭、关系与财务信息默认不进入公开输出。
- 客户、用户和合作资料默认 internal，并应匿名化。
- AI 新提取资产默认“待确认”。
- 结构写入前先展示预案；已有文件默认跳过。

## 用户责任

- 在已有 Vault 使用前先备份。
- 处理客户、同事或第三方资料前取得必要授权。
- 不要把 API Key、Token、Cookie、密码或身份证件写入 Vault。
- 公开内容前人工检查事实、隐私与版权。
- 不要把 `.obsidian/workspace*.json`、私人录音和真实客户资料提交到公开仓库。

## 删除与暂停

采访可以随时暂停。用户可以要求删除当前问题的回答；Agent 应在继续派生前完成本地删除或标记，并说明哪些已生成笔记需要同步处理。
