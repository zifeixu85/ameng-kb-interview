# 知识库说明

## 核心主张

知识库不是文件夹收藏，而是一套让过去的资料继续参与判断、项目和输出的系统。

## 固定职责

```text
00-系统工作台  规则、模板、采访来源与结构配置
10-自我说明书  真实自我、经历、价值观、状态、偏好、关系、AI 协作与隐私边界
20-可复用资产库  从经历和输入中提取的可复用资产
30-项目工作区  有目标、周期、里程碑和上下文的项目
40-学习输入库  尚待消化的外部输入与主题学习
99-附件库  图片、音频、PDF 等附件
```

`20` 和 `30` 下面的具体模块由采访结果决定。没有真实产品、客户、内容或黑客松经历时，就不创建相应目录。

`10-自我说明书` 采用完整的十维地图，但也按证据渐进创建：身份经历、价值观驱动力、性格心理、偏好审美表达、健康能量、家庭亲密关系、社会关系、阶段复盘、日记观察、AI 协作偏好。敏感模块必须明确同意后启用；真实自我不能直接等同于对外人设。

## 一条资料如何进入系统

```text
输入 → 保留来源 → 提取理解 → 形成待确认资产 → 在项目或输出中使用 → 写回反馈
```

正式资产按主题和用途组织，来源写进笔记属性，不按书名、课程名或老师姓名建立资产目录。

## 状态与敏感级别

- AI 新提取的资产：`status: 待确认`
- 用户确认可持续使用后：`status: 可复用`
- 私密自我：`sensitivity: private`
- 工作内部资料：`sensitivity: internal`
- 用户明确授权公开：`sensitivity: public`

目录位置不能替代状态和隐私判断。
