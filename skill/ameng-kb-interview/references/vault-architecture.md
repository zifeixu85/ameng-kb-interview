# Vault 架构：固定骨架 + 动态模块

## 设计目标

第一版知识库应当足够小，但能持续生长。固定目录负责稳定职责；动态目录只在采访发现真实场景后创建。

## 固定最小骨架

```text
Vault/
├── AGENTS.md
├── 00-系统工作台/
│   ├── 知识库说明.md
│   ├── 建库配置.md
│   ├── 采访记录/
│   └── 模板/
├── 10-自我说明书/
│   ├── 01-身份与经历/
│   ├── 02-价值观与判断/
│   ├── 03-能力与偏好/
│   └── 99-隐私边界与公开授权/
├── 20-可复用资产库/
│   └── README.md
├── 30-项目工作区/
│   └── 项目总览.md
├── 40-学习输入库/
└── 99-附件库/
    └── _inbox/
```

### 顶层职责

| 目录 | 负责什么 | 不负责什么 |
|---|---|---|
| `00-系统工作台` | 规则、模板、采访原文、建库配置 | 业务资产与长期内容 |
| `10-自我说明书` | 真实自我、经历、偏好、隐私边界 | 默认公开表达 |
| `20-可复用资产库` | 已提取的角色、故事、案例、方法、输出资产 | 未消化的原始资料 |
| `30-项目工作区` | 项目目标、状态、关键判断、里程碑、复盘 | 每一个零散任务 |
| `40-学习输入库` | 待消化的课程、书、文章、活动与主题学习 | 已确认的方法论 |
| `99-附件库` | 图片、音频、PDF 等附件 | Markdown 知识正文 |

## 动态可选模块

只有采访证实用户会持续使用时才创建。

| 模块 ID | 目录 | 触发证据 |
|---|---|---|
| `profile` | `20-可复用资产库/1-对外角色与表达` | 对外介绍、公开写作或个人品牌是持续场景 |
| `products` | `20-可复用资产库/2-产品与服务` | 正在提供产品、课程、咨询或服务 |
| `customers` | `20-可复用资产库/3-客户与用户` | 有客户、用户访谈、异议或需求证据 |
| `stories` | `20-可复用资产库/4-素材与案例` | 有值得复用的经历、案例、事实或原话 |
| `methods` | `20-可复用资产库/5-方法与判断` | 已形成可复述、可验证的判断或做事方法 |
| `content` | `20-可复用资产库/6-内容生产` | 稳定写作、视频、播客、社媒或课程输出 |
| `business_knowledge` | `20-可复用资产库/7-业务知识` | 有重复 FAQ、SOP、服务流程或协作规则 |
| `opportunities` | `20-可复用资产库/8-合作与机会` | 需要跟进线索、合作、报价或承诺 |
| `talks` | `20-可复用资产库/9-主题分享` | 经常演讲、授课、直播或复讲主题 |
| `career` | `20-可复用资产库/10-职业资产` | 求职、晋升、作品集、绩效或能力证明是核心目标 |

`content` 模块默认包含 `选题/`、`母稿/`、`发布版本/`、`发布复盘/`；`business_knowledge` 包含 `FAQ/`、`SOP/`、`服务流程/`。其他模块只创建入口 README，且必须同时生成至少一篇真实笔记，不能只留下空壳。

## 动态项目类型

项目文件夹按稳定类型分，不按“进行中 / 已结束”分。状态写进项目母舰。

| 类型 ID | 目录 | 何时创建 |
|---|---|---|
| `work` | `30-项目工作区/工作项目` | 有跨周的岗位项目或内部项目 |
| `client` | `30-项目工作区/客户交付` | 有明确客户、范围、交付或验收 |
| `product` | `30-项目工作区/产品实验` | 正在做产品、MVP 或功能验证 |
| `content` | `30-项目工作区/内容项目` | 连载、栏目、课程或大型内容项目 |
| `learning` | `30-项目工作区/学习项目` | 有目标、周期与成果物的学习计划 |
| `community` | `30-项目工作区/社群与活动` | 持续运营社群或活动项目 |
| `hackathon` | `30-项目工作区/黑客松项目` | 用户明确参加过或正在参加黑客松 |

满足以下任一条件才建项目包：持续超过 7 天；需要两次以上工作会话；有里程碑、截止日期、对外承诺或多份过程资料。

## 不进入公开初始结构的目录

不要默认创建：

- 历史采矿场；
- 原始资料旧库；
- 归档冷库；
- 黑客松项目；
- 客户、产品、内容或商机目录。

这些都是特定用户或迁移阶段的需要，不是个人知识库的通用前提。

## 已有 Vault 接入

1. 先列出现有顶层目录与关键入口文件。
2. 先做职责映射，不立即重命名或移动。
3. 若已有等价目录，在结构方案中记录映射并复用。
4. 新文件只写入已确认的目标路径。
5. 任何迁移、合并、删除必须成为单独任务并再次确认。
