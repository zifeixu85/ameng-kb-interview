# 采访执行手册

## 采访方式

- 一次只问一个问题。
- 每个阶段先说明“这一段是为了决定什么”。
- 第一个正式问题先认识用户，不要从“你想让知识库做什么”开问。
- 询问用途、输入、输出或模块时，优先根据前文给出 2–3 个具体候选，不要让用户理解抽象分类后再回答。
- 选项是启发，不是限制。每次保留“都不是，我自己说”和“还不确定，请你推荐”。
- 用户已经明确说出用途、真实材料或目标产物时，直接复用这些答案，不重复询问。仍先保留 1–2 个认识用户的问题，然后直接确认或微调闭环。
- 用户回答抽象时，从“最近一次”“当时怎么做”“产生了什么结果”“你原话怎么说”中选一个追问。
- 用户没有相关经历时，直接跳过分支。
- 不诱导用户编造产品、客户、内容习惯或项目。

## 阶段 0：先认识用户，再确定首个闭环

目标：先理解用户当下的身份、重复能力和真实摩擦，再帮他选出第一条使用闭环。Vault 路径、新旧库和采访深度已在采访前确认，不占用正式题号。

1. **认识当下**：先不谈知识库。你现在主要在做什么？如果不好概括，就说说最近一周大部分时间花在哪里。
2. **认识重复能力**：别人通常因为什么事情来找你，或者哪类事情你已经做过不止一次？
3. **定位真实摩擦**：最近一个月，哪种时刻最容易让你觉得“下次不想再从头找、从头想、从头说”？

第 3 题给出与前文最相关的 2–3 个选项。无法从前文判断时，再从以下示例中选最接近的 3 个：

- 项目中断后，要重新找背景和决策；
- 学了很多，到真正要用时找不到；
- 写内容时，每次都要重新找素材和想选题；
- 同一个问题、流程或服务反复向不同人解释；
- 项目、客户或产品信息散在聊天和文件里；
- 不确定，想先说一次最近的真实例子。

如果用户在开场已经说明了真实摩擦，跳过第 3 题的通用选项，把用户原话作为已确认答案。

4. **提出闭环候选**：基于前三题，AI 先复述一句已经理解的上下文，再提出最多 3 个闭环候选。每个候选都用这个格式：

```text
A. 真实输入 → AI 处理 → 具体产物 → 下次如何复用
```

然后只问：“今天你最想先跑通哪一个？也可以说都不是，或让我根据频率和当下价值推荐一个。”

如果用户已经给出明确用途和真实材料，不再给 3 个泛化候选；只给 1 个建议闭环和最多 1 个轻量备选，让用户确认或修改。

5. **锁定真实输入**：针对选中的闭环，今天可以拿哪一份真实材料或哪个具体例子来跑？

输入选项必须与选中闭环相关。用户没有文件时，允许他口述最近一次真实实例，或直接使用本次采访原话。

6. **定义第一次完成**：用这份材料，你希望第一版具体拿到什么？怎样才算真的帮上忙？

先给出 2–3 个符合所选闭环的产物选项，例如项目恢复卡、内容选题与初稿、学习消化卡与实验、脱敏 FAQ/SOP、对外介绍或方法卡。

7. **确认本次边界**：只针对刚才选中的材料，哪些信息只能帮助 AI 理解你，不能进入共享或对外内容？

快速版回答到能确定“一个真实输入 + 一个具体产物 + 一个下次复用场景 + 本次隐私边界”即可收口。只针对选中闭环再追问最多 1–2 题，然后直接进入结构方案确认；不按顺序跑完后面的完整版阶段。完整版继续进入阶段 1–5。

## 阶段 1：身份、经历与边界

目标：建立 private 自我上下文和 internal 对外表达，二者分开。

1. 如果从今天往回看，你经历过哪几个对现在最重要的阶段？
2. 这些阶段分别让你获得了什么能力、资源或判断？
3. 别人通常因为什么事情来找你？
4. 你做选择时最看重什么？有哪些事情即使有机会也不会做？
5. 哪些经历或信息只能用于帮助 AI 理解你，绝不能进入公开内容？

输出候选：身份经历、能力地图、价值观与判断、隐私边界。只有用户会持续对外介绍或公开表达时，才启用 `profile` 并生成对外角色草稿。

## 阶段 2：项目与行动上下文

目标：识别哪些事情值得建项目母舰，而不是把所有待办塞进知识库。

1. 现在有哪些事情会持续一周以上，或需要多次推进？
2. 其中哪一件有明确截止日期、对外承诺、交付物或合作方？
3. 如果一个月后重新接手，你最怕忘掉哪些背景、决策和证据？
4. 你现在用什么管理下一步行动和固定日程？如果没有，也不需要为了建库新增工具。

根据答案选择项目类型。只有用户明确提到比赛或黑客松时才启用 `hackathon`。

## 阶段 3：学习与输入

目标：建立输入管道，而不是把收藏夹复制进 Vault。

1. 最近三个月，你真正学完或反复使用过哪些主题？
2. 一份外部资料要经过什么过程，才算变成“你会用的东西”？
3. 你通常如何记录来源、自己的理解和下一步试验？
4. 哪一种输入最值得先做成固定处理流程？

输出候选：学习主题地图、资料卡、消化笔记、第一条输入处理规则。

## 阶段 4：故事、案例与方法

目标：把经历提取成可追溯、可复用的资产，不把整段人生压成一篇简介。

1. 最近一年，哪一次具体经历最能说明你如何解决问题？
2. 当时的背景、行动、结果和证据分别是什么？
3. 哪一次失败或变化让你后来形成了一条新的做事规则？
4. 你有哪些经常重复使用的判断步骤、检查清单或解释方式？

每个故事和每条方法分别写成独立笔记；保留事实、原话和待确认项。确实提取出可复用故事时启用 `stories`，确实形成方法或判断时启用 `methods`。

## 阶段 5：条件分支

先问：“下面哪些场景是你未来三个月会持续使用的？”只进入被选择的分支。

### 内容输出（启用 `content`）

1. 你稳定输出到哪些平台或载体？
2. 一篇内容通常从什么素材开始，到什么状态才算完成？
3. 你希望 AI 读取哪些资料，又必须避开哪些资料？

### 产品与服务（启用 `products`）

1. 你当前真正提供的产品或服务是什么？
2. 谁会使用、解决什么问题、如何判断有效？
3. 哪些说法已经验证，哪些仍然只是设想？

### 客户与用户（启用 `customers`）

1. 哪些用户问题、异议或需求被重复提到？
2. 现有证据来自访谈、聊天记录还是实际使用？
3. 哪些信息涉及真实客户，必须匿名或限制读取？

### FAQ、SOP 与服务流程（启用 `business_knowledge`）

1. 哪些问题或流程说明被重复回答、重复执行？
2. 哪些部分可以标准化，哪些必须人工判断？
3. 现有证据来自 FAQ、SOP、会议、聊天记录还是实际交付？

### 商机与合作（启用 `opportunities`）

1. 哪些合作需要持续跟进承诺、资料或下一次联系？
2. 哪些信息只属于关系管理，不应进入公开案例？

### 主题分享（启用 `talks`）

1. 哪些主题可能反复讲、持续更新？
2. 一场分享结束后，哪些内容需要反向沉淀成素材、方法和内容？

### 职业资产（启用 `career`）

1. 你需要为求职、晋升或作品集证明哪些能力？
2. 哪些项目和结果可以成为证据？

## 阶段 6：结构确认与首个闭环

1. 展示固定骨架和动态模块。
2. 对每个动态模块说明触发它的原始回答。
3. 让用户删除不想维护的模块，而不是鼓励“以后可能用”。
4. 共同选定采访结束后立即跑通的一次真实输出。

## 快速版最低问题集

快速版是独立路线，不是完整版问卷的缩减版。只需问清：

1. 用户当下在做什么；
2. 他重复做过的能力型事情；
3. 最近一次真实摩擦；
4. AI 提出的最多 3 个闭环候选中，今天先跑哪一个；
5. 一份真实输入和一个具体产物；
6. 本次材料的隐私与对外边界。

项目、故事、方法和动态模块由前面的真实回答触发，不为了凑完整而强问。多个候选并存时，优先选择：眼前有真实材料、近期会再次发生、最快能产出可用结果的一个。
