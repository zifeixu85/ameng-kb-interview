# 写入路由与隐私规则

> 本文的路径是逻辑标准路径。已有 Vault 必须先应用经用户确认的 `existing_mappings`，再读写实际目录。

## 三种敏感级别

| 值 | 含义 | 默认使用边界 |
|---|---|---|
| `private` | 只用于理解用户本人 | 不进入公开输出，不跨库复制细节 |
| `internal` | 可用于工作与内容准备 | 对外发布前仍需检查 |
| `public` | 用户明确授权公开复用 | 可进入对外输出流程 |

身份、日记、健康、家庭、关系、财务压力默认 `private`。产品、客户、合作信息默认 `internal`。只有用户明确确认后才使用 `public`。

## 回答路由

| 回答内容 | 默认路径 | 默认敏感级别 |
|---|---|---|
| 完整经历、内在驱动力、个人限制 | `10-自我说明书` | `private` |
| 可公开身份标签、介绍、表达偏好 | `20-可复用资产库/1-对外角色与表达` | `internal` |
| 具体故事、案例、原话、结果 | `20-可复用资产库/4-素材与案例/<主题>` | `internal`；涉及他人时脱敏 |
| 判断框架、检查清单、做事方法 | `20-可复用资产库/5-方法与判断/<主题>` | `internal` |
| 产品与服务 | 可选模块 `2-产品与服务` | `internal` |
| 客户原话、问题、异议、FAQ | 可选模块 `3-客户与用户` | `internal`；默认匿名 |
| 选题、母稿、发布记录 | 可选模块 `6-内容生产` | `internal`，发布快照可 `public` |
| FAQ、SOP、服务流程 | 可选模块 `7-业务知识` | `internal`；标记人工判断边界 |
| 线索、报价、合作承诺 | 可选模块 `8-合作与机会` | `internal` |
| 分享主题、讲稿、场次复盘 | 可选模块 `9-主题分享` | `internal` |
| 求职、晋升、作品集证据 | 可选模块 `10-职业资产` | `internal` |
| 项目目标、决策、里程碑、复盘 | `30-项目工作区/<类型>/<项目>/项目总览.md` | `internal` |
| 未消化课程、文章、书、录音 | `40-学习输入库/<主题>` | `internal`，观点归属未确认 |

## 同一回答的分拆

当一段回答同时含 private 自我经历和可公开表达时，拆成两篇：

1. `10-自我说明书` 保留完整上下文；
2. `20-可复用资产库/1-对外角色与表达` 只保留用户愿意对外使用的版本。

两篇可以互链，但对外笔记不得复述 private 细节。

## 原料与观点

- 采访原文是来源，不自动等于用户认可的观点。
- 外部课程和文章是输入，不自动写成用户方法论。
- AI 提取的新判断默认 `status: 待确认`。
- 涉及第三方姓名、联系方式、合同金额、健康和关系信息时，公开示例使用虚构或匿名内容。

## 写入确认

以下动作可以在结构方案一次确认后执行：创建缺失目录、写入采访原文、生成待确认笔记、建立双链。

以下动作必须再次确认：

- 覆盖、移动、重命名或删除已有文件；
- 把 `private` 内容改为 `internal/public`；
- 把 `待确认` 改为 `可复用`；
- 对外发布、给客户发消息、提交表单；
- 创建带承诺、付款、客户沟通的任务或日历事件。
