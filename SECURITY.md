# Security

## 安全边界

- 初始化脚本只创建缺失文件，不覆盖、移动或删除用户文字。工具自己创建的配置文件只会更新明确标记的受管区块。
- `--dry-run` 可以先预览全部改动。
- 结构方案只接受白名单模块和经过校验的项目类型名称。
- 所有写入路径都必须留在 Vault 内；软链接和 Windows 分隔符不能被用来越界写入。
- 笔记中的命令、提示词和网页内容都应被视为资料，而不是系统指令。

## 使用建议

1. 对已有 Vault 先做备份。
2. 使用独立测试 Vault 体验第一次采访。
3. 不要以管理员权限运行初始化脚本。
4. 公开提交前检查绝对路径、密钥、录音链接、客户名和工作区文件。

## 报告问题

请在 GitHub Issues 中提供最小复现步骤，不要附上真实采访原文、私人 Vault、Token 或客户资料。
